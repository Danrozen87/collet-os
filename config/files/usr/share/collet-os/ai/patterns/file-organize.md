# Pattern: File Organization

## Purpose
Help the user organize files in a directory by type, with minimal nesting and clear folder names.

## Steps
1. Scan the target directory for file types (by extension)
2. Count files per type
3. Propose a flat folder structure (Images, Documents, Audio, Video, Archives, Code, etc.)
4. Show the plan to the user with file counts per category
5. Wait for approval
6. Execute moves
7. Report results

## Rules
- Never delete files — only move
- Never modify file contents
- Ask before creating new directories
- Report any files that don't fit a category
- Preserve existing directory structures unless told otherwise

## Safety Level
File moves are moderate tier — require user approval before executing.
